# **5. 실제 MVP 개발에 바로 연결되는 질문**

현재 문서 기준으로 MVP는 **유사 사례 검색 → 작업지시서 초안 → 원가·마진 검토 → 누락 사양 질문 → 승인본 저장** 흐름이 가장 적합합니다. 이를 검증하려면 아래 질문이 중요합니다.

66. 제품 사진이나 설명만 받았을 때, 사람이 작업지시서 초안을 만들기 위해 추가로 묻는 질문은 무엇인가요?
67. 작업지시서 초안을 만들 때 최소 입력값은 무엇인가요?  
    예: IP명, 상품 종류, 사이즈, 수량, 소재, 색상, 납기, 판매처, 목표가.
68. AI가 초안을 만든 뒤 사람이 가장 먼저 검토할 부분은 어디인가요?
69. “누락 사양 질문”을 AI가 자동으로 한다면, 반드시 물어봐야 할 질문 리스트는 무엇인가요?
70. 유사 제작사례 검색 결과에서 보여줘야 할 정보는 무엇인가요?  
    예: 상품 사진, 사양서, 견적서, 제조사, 단가, MOQ, 납기, 이슈, 최종 판매가, 재발주 여부.
71. 과거 사례를 기반으로 예상 원가를 제안할 때 허용 가능한 오차 범위는 어느 정도인가요?
72. AI가 추천한 제조사나 원가를 사람이 수정하면, 그 수정 사유를 기록해야 하나요?
73. 승인된 최종본은 어디에 저장되어야 하나요?  
    ERP, 원드라이브, 신규 시스템, 또는 모두 연동.
74. 승인 완료 후 수정이 발생하면 새 버전으로 관리해야 하나요, 기존 문서를 덮어써도 되나요?
75. MVP에서 가장 먼저 자동화하면 체감 효과가 큰 기능 1개는 무엇인가요?  
    후보: 과거 사례 검색, 작업지시서 초안, 견적 비교, 품질표시사항 초안, 제조사 추천, 일정 지연 알림.

이 부분이 가장 중요합니다. 문서에는 “과거 상품 개발 경험, 제조사·거래처별 특성, 과거 품질 이슈 대응 경험이 담당자 경험에 의존한다”고 되어 있습니다. 이 암묵지를 뽑아야 온톨로지가 실제로 쓸모 있어집니다.  

76. 특정 제조사는 어떤 품목을 잘하고, 어떤 품목은 피해야 하나요?

80. 원가 절감할 때 절대 건드리면 안 되는 요소는 무엇인가요?

84. 납기 지연이 예상될 때 가장 먼저 확인해야 하는 신호는 무엇인가요?
85. “이건 경험상 안 된다”라고 판단하는 대표 사례를 3개만 들어주세요.

# **7. 개발자인 내가 인터뷰 후 정리해야 할 산출물**

면담이 끝나면 단순 회의록이 아니라 아래 형태로 정리해야 합니다.

## **1차 산출물: 온톨로지 후보 객체**

- IP
- LicenseContract
- ProductProject
- Product
- SKU
- ProductSpec
- Quote
- Vendor
- Sample
- Approval
- PurchaseOrder
- ProductionBatch
- Certification
- LabelingInfo
- Inbound
- Inventory
- SalesChannel
- Issue
- Claim
- DecisionLog
- Document
- CommunicationLog

## **2차 산출물: 핵심 관계**

- IP는 여러 ProductProject에 연결된다.
- ProductProject는 여러 Product/SKU를 가진다.
- SKU는 ProductSpec, Quote, Sample, Certification, PurchaseOrder와 연결된다.
- Vendor는 Quote, Sample, ProductionBatch, Issue 이력을 가진다.
- Approval은 기획안, 견적, 샘플, 발주, 출시 단계에 연결된다.
- Document는 모든 객체의 근거 자료가 된다.
- Issue는 Vendor, SKU, ProductionBatch, Claim과 연결된다.

## **3차 산출물: AI 기능 후보**

- 유사 제작사례 검색
- 작업지시서 초안 생성
- 견적서 필드 자동 추출
- 제조사 추천
- 원가·마진 자동 계산
- 누락 사양 질문 생성
- 품질표시사항 초안 생성
- 인증 필요 여부 체크
- 양산 전 위험 체크리스트
- 승인본·최종본 자동 식별
- 프로젝트별 의사결정 히스토리 요약
- 담당자 변경 시 인수인계 요약
