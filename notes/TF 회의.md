# 06 11
---

회의 핵심은 **“이미지 AI 자체를 만들자”가 아니라, 내부 업무 데이터를 구조화해서 유사 사례·원가·소재·계약·제조사 정보를 바로 찾게 만드는 시스템을 만들자**로 정리됩니다.

**회의 요약**

1. **AI 전환 프로젝트 방향**
    
    - 각 팀 업무를 인터뷰해서 업무 흐름을 플로우차트로 정리한다.
        
    - 그중 AI를 넣을 수 있는 지점을 찾고, 일부 상품/프로세스에 먼저 적용한다.
        
    - 효과가 있으면 대표 직속팀 → 전사 적용 순서로 확장한다.
        
2. **데이터 라벨링은 별도 작업이 아니라 병행**
    
    - 작업지시서, 견적서, 계약서, 원가, 소재, 제조사, 제품 사진 등을 모아야 한다.
        
    - OCR로 문서 내용을 뽑고 JSON/구조화 데이터로 만든 뒤, 사람이 검수·라벨링해야 한다.
        
    - 어떤 항목이 필요한지 개발자가 혼자 판단하기 어렵기 때문에 실무자 인터뷰에서 기준을 뽑아야 한다.
        
3. **핵심 MVP 방향**
    
    - 제품/IP/소재/제조사/계약서/원가/디자인 에셋을 서로 연결하는 형태가 적합하다는 의견.
        
    - 예: 특정 IP나 제품을 선택하면 관련 유사 사례, 과거 원가, 계약 내용, 제조사, 소재, 디자인 자료가 같이 보이는 방식.
        
    - 단순 검색이 아니라 “강한 연결 / 보통 연결 / 약한 연결”처럼 연관도 기반으로 확장해서 볼 수 있는 구조가 좋다고 평가됨.
        
4. **디자인 AI 자체 개발은 우선순위 낮음**
    
    - 디자이너의 문제는 “디자인을 못해서”가 아니라, 생산·소재·원가·원청 피드백 과정에서 반복 수정과 판단 부담이 커지는 것.
        
    - 따라서 자체 이미지 생성 AI를 만드는 것보다, 기존 상용 이미지 AI를 쓰고 내부 데이터 기반 업무 판단 시스템을 만드는 게 더 현실적.
        
    - 집중해야 할 것은 **휴먼 에러와 휴먼 딜레이 감소**.
        
5. **티켓팅/팬덤 플랫폼 현황**
    
    - 현재 티켓팅 플랫폼은 작동 중이며 결제 이슈는 해결됨.
        
    - 다만 판매가 기대보다 낮고, 취소표가 많음.
        
    - 손익분기점은 약 2억 매출로 언급됐고, 현재는 약 1.8억 수준으로 이야기됨.
        
    - 행사 전 홍보 영상 등을 통해 2억까지는 회복 가능성이 있지만, 큰 폭 성장은 어렵다고 봄.
        
    - 수집 가능한 데이터: 이름, 이메일, 전화번호, 국가, 티켓 구매 내역, 티어별 지출, 결제수단, 마케팅 동의 여부 등.
        
6. **팬덤 데이터의 한계**
    
    - 현재 티켓팅 팬덤은 특정 서브컬처 성향이 강해, 아이콘스의 일반 IP 팬덤 플랫폼으로 자연스럽게 전환될 가능성은 낮다는 의견.
        
    - 향후 IP 플랫폼에서는 회원가입 시 선호 IP를 선택하게 해서 취향 데이터를 수집하는 방식이 필요하다는 제안이 나옴.
        
7. **개발 협업 방식**
    
    - AI를 활용한 개발은 각자가 동시에 작업할 경우 중복 구현이나 충돌이 생길 수 있음.
        
    - GitHub 기반으로 현재 진행 상황을 먼저 파악하고, 칸반/Markdown 체크리스트 방식으로 작업 관리하는 방향이 언급됨.
        
    - 최소 며칠 정도는 AI 협업 개발 방법론을 맞추는 시간이 필요하다는 의견.
        

**결정/합의된 방향**

- 대표 직속팀 인터뷰를 진행해 실제 업무 흐름과 데이터 위치를 파악한다.
    
- 기존 클라우드에 흩어진 업무 자료를 일단 모으고, 이후 업로드/파일명/폴더/버전 관리 가이드를 만든다.
    
- 디자인 AI 자체 개발보다는 내부 데이터 라벨링 + 유사 사례 검색 + 원가/소재/계약 판단 보조에 집중한다.
    
- IP 플랫폼은 단순 커뮤니티보다 고객 선호 IP 데이터 수집 구조가 중요하다.
    
- 티켓팅 플랫폼의 고비용 실시간 인프라는 현재 AI/IP MVP에는 그대로 가져오지 않는 것이 맞다.
    

**액션아이템**

- 예린 님 인터뷰 진행: 업무 프로세스, 데이터 저장 위치, 산출물 종류, 반복 피드백 구간 파악.
    
- 대표 직속팀 클라우드 접근 권한 확보.
    
- 기존 클라우드 자료 전체 수집 후, 향후 업로드 규칙/폴더 규칙/버전 관리 기준 설계.
    
- 작업지시서·견적서·계약서·원가표의 라벨링 기준 초안 작성.
    
- IP/제품/소재/제조사/계약/원가/디자인 에셋 연결 구조 시각화 MVP 구체화.
    
- 개발 담당자끼리 GitHub 현황 공유 후 AI 협업 개발 방식 정리.
    
- 티켓팅 플랫폼 고객 데이터 중 향후 IP 플랫폼에 재사용 가능한 항목 검토.
    

한 줄로 정리하면, 이번 회의는 **“회사 내부 데이터를 AI가 이해할 수 있는 구조로 바꿔서, 실무자가 과거 사례·원가·소재·계약 판단을 빠르게 하게 만드는 방향으로 가자”**는 회의입니다.
