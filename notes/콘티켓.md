## KONticket 결제 페이지 기술스택 분석

현재 페이지의 결제 방식 섹션을 분석한 결과입니다.[kon-ticket](https://kon-ticket.com/bookticket/pay/30e2193e-d714-4a91-94a4-8a4d1a821c10)

## 핵심 결제 게이트웨이: **Omise (오마이즈)**

페이지 소스의 라디오 버튼 aria-label에서 세 가지 결제 수단 모두 `OMISE` 접두사가 명확히 표기되어 있습니다:[kon-ticket](https://kon-ticket.com/bookticket/pay/30e2193e-d714-4a91-94a4-8a4d1a821c10)

text

`credit_card-OMISE promptpay-OMISE truemoney_jumpapp-OMISE`

즉, KONticket은 **Omise**를 단일 결제 게이트웨이로 사용하여 세 가지 결제 수단을 모두 통합 처리합니다.

---

## 결제 수단별 기술스택 상세

## 1. Credit/Debit Card (신용카드/직불카드)

|항목|내용|
|---|---|
|처리 게이트웨이|Omise Payment API|
|지원 카드 네트워크|Visa, Mastercard, Amex, UnionPay, JCB|
|보안 방식|PCI-DSS 준수, 카드 정보 토큰화(Tokenization)|
|입력 방식|카드번호 → 만료일 → CVV → Omise API로 토큰 발급 후 서버 전송|

Omise는 카드 정보를 직접 서버로 전송하지 않고 **클라이언트 사이드 토큰화(Client-side tokenization)** 방식을 사용합니다.[github](https://github.com/omise/omise-ios)

## 2. PromptPay QR

|항목|내용|
|---|---|
|처리 게이트웨이|Omise → PromptPay 연동|
|기반 인프라|태국 중앙은행(BOT, Bank of Thailand)이 운영하는 국가 전자결제 시스템|
|기술 방식|Thai QR Standard(EMVCo QR Code 기반), 실시간 계좌이체|
|결제 흐름|Omise API → QR 생성 → 사용자 은행앱 스캔 → 실시간 정산|

PromptPay는 태국의 국가 인프라로, 6백만 이상의 가맹점이 사용하는 표준화된 QR 결제 시스템입니다.[fastpayments.worldbank](https://fastpayments.worldbank.org/sites/default/files/2021-09/World_Bank_FPS_Thailand_PromptPay_Case_Study.pdf)

## 3. TrueMoney (truemoney_jumpapp)

|항목|내용|
|---|---|
|처리 게이트웨이|Omise → TrueMoney Wallet 연동|
|서비스 주체|Ascend Money (태국 최대 핀테크, CP그룹 계열)|
|기술 방식|딥링크(Deep Link) / Jumpapp 방식으로 TrueMoney 앱으로 리다이렉트|
|결제 흐름|Omise API → `payment_url` 발급 → TrueMoney 앱 실행(jumpapp) → 결제 승인 → 콜백|

`truemoney_jumpapp`이라는 식별자에서 앱 간 이동(Jumpapp) 방식을 사용하는 것을 확인할 수 있습니다.airwallex+1

## 전체 결제 아키텍처 요약

text

`KONticket 프론트엔드         ↓   Omise API (단일 게이트웨이)   ├── Credit/Debit Card → 카드 토큰화 처리   ├── PromptPay QR      → BOT 국가결제망 연동   └── TrueMoney         → Ascend Money 앱 딥링크 연동`

- **통화**: THB (Thai Baht) — 태국 현지 결제 시스템에 최적화된 구성
    
- **게이트웨이**: Omise는 태국/동남아시아 특화 결제 서비스로, 카드·QR·전자지갑을 하나의 API로 통합 제공[stackshare](https://stackshare.io/omise)
    
- **보안**: PCI-DSS, 클라이언트 사이드 토큰화로 카드 정보를 직접 서버에 전달하지 않는 구조[bitscale](https://bitscale.ai/directory/omise)
    
- **SDK 지원**: Omise는 웹(JavaScript), iOS, Android SDK를 제공하여 멀티플랫폼 통합 가능[github](https://github.com/omise/omise-ios)
