# 모든 것에 포스트그레스를 사용하세요

https://www.tigerdata.com/blog/its-2026-just-use-postgres
- 검색 → pg_textsearch (BM25) → Elasticsearch 대체
- 벡터 검색 → pgvector + pgvectorscale (DiskANN) → Pinecone보다 28배 빠르고 75% 저렴
- 시계열 → TimescaleDB → InfluxDB보다 비슷하거나 우수 + 전체 SQL 지원
- 문서 → JSONB → MongoDB급 성능 + ACID 보장
- 지리정보 → PostGIS (2001년부터 표준)
- 큐 → pgmq → Kafka 대체 가능
- 그 외 pg_cron, pgai 등으로 대부분 커버


## 한국어 버전
## **그냥 Postgres를 쓰세요 (Just Use Postgres)**

Redis 대신 캐싱으로 사용

UNLOGGED 테이블과 JSON 데이터 타입용 TEXT를 사용하세요. 저장 프로시저(Stored Procedures)를 직접 작성하거나, 저처럼 ChatGPT를 활용해 작성해 보세요. Redis처럼 데이터 만료 기한을 추가하고 강제할 수 있습니다.

Kafka 대신 메시지 큐로 사용

메시지 큐 기능만 필요하다면 SKIP LOCKED 기능을 사용하세요. Go 언어 환경에서 작업 큐(Job Queue)가 필요하다면 River를 추천합니다.

데이터 웨어하우스(Data Warehouse)

Timescale을 함께 사용하여 데이터 웨어하우스로 활용하세요. S3 연동이 필요하다면 DuckDB를 사용하면 됩니다.

데이터 레이크(Data Lake)

Postgres와 DuckDB를 조합해 데이터 레이크를 구축하세요. 이를 위한 DuckLake라는 제품도 있으며, Postgres를 카탈로그로 사용합니다.

인메모리 OLAP

Apache Datafusion 기반의 pg_analytics를 사용하여 Postgres에서 인메모리 OLAP를 구현하세요.

MongoDB 대신 문서 데이터베이스로 사용

JSONB를 사용하여 JSON 문서를 저장, 검색 및 인덱싱하세요. 더 완벽한 교체 옵션을 원한다면 DocumentDB를 사용해 보세요.

크론 데몬(Cron Daemon)

pg_cron을 사용해 특정 시간에 이메일 발송 등의 작업을 수행하거나 메시지 큐에 이벤트를 추가하는 크론 데몬으로 활용하세요.

지리 정보(Geospatial) 쿼리

PostGIS를 사용하여 지리 정보 쿼리를 처리하세요.

Elasticsearch 대신 전체 텍스트 검색(Full-text Search)

전문 검색(Full-text Search) 엔진 대신 Postgres 자체 기능을 사용하세요.

JSON API 생성

서버측 코드를 작성하는 대신 데이터베이스에서 직접 JSON을 생성하여 API로 바로 전달하세요.

감사(Auditing)

pgaudit 확장 프로그램을 사용하여 데이터베이스 감사 기능을 구현하세요.

GraphQL 서비스

필요한 경우 GraphQL 어댑터(GraphJin)를 사용하여 Postgres에서 바로 GraphQL을 제공하세요.

벡터 데이터베이스 (Vector Database)

특화된 벡터 DB 대신 pgvector를 사용하여 임베딩 및 AI 유사도 검색을 수행하세요.

세션 스토어 (Session Store)

Redis 대신 세션 ID와 만료 시간에 적절한 인덱스를 설정한 Postgres를 사용하세요. 키-값(Key-Value) 저장소가 필요하다면 hstore를 활용하세요.

처리율 제한 (Rate Limiting)

Redis 카운터 대신 원자적 업데이트(Atomic updates)와 시간 기반 윈도우를 사용하는 Postgres 기반 레이트 리미팅을 도입하세요.

**Just Use Postgres**

**Caching instead of Redis**

Use Postgres with [**UNLOGGED tables**](https://www.compose.com/articles/faster-performance-with-unlogged-tables-in-postgresql/) and TEXT as a JSON data type. [**Use stored procedures**](https://www.postgresql.org/docs/current/xproc.html) or do as I do, use [**ChatGPT**](https://chat.openai.com/chat) to write them for you, to add and enforce an expiry date for the data just like in Redis.

**Message queue instead of Kafka**

Use Postgres as a message queue with [**SKIP LOCKED**](https://www.2ndquadrant.com/en/blog/what-is-select-skip-locked-for-in-postgresql-9-5/) (if you only need a message queue). Or as a job queue in Go with [**River**](https://brandur.org/river)

**Data warehouse**

Use Postgres with [**Timescale**](https://www.timescale.com/) as a data warehouse. Use [**DuckDB for S3 integration**](https://duckdb.org/docs/stable/guides/network_cloud_storage/s3_import.html).

**Data lake**

Use Postgres together with [**DuckDB**](https://duckdb.org/docs/stable/core_extensions/postgres.html) as a data lake. There is a product for this called [**DuckLake**](https://ducklake.select/), with Postgres as the catalog.

**In-memory OLAP**

Use Postgres with [**pg_analytics**](https://docs.paradedb.com/blog/introducing_analytics) as an in memory OLAP [**with Apache Datafusion**](https://arrow.apache.org/datafusion/)

**Document database instead of MongoDB**

Use Postgres with [**JSONB**](https://scalegrid.io/blog/using-jsonb-in-postgresql-how-to-effectively-store-index-json-data-in-postgresql/) to store Json documents in a database, search and index them. If you need a more drop-in like replacement, use [**DocumentDB**](https://github.com/documentdb/documentdb).

**Cron daemon**

Use Postgres as a cron demon to take actions at certain times, like sending mails, with [**pg_cron**](https://github.com/citusdata/pg_cron) adding events to a message queue.

**Geospatial queries**

Use Postgres for [**Geospatial queries**](https://postgis.net/).

**Full-text search instead of Elastic**

Use Postgres for [**Fulltext Search**](https://supabase.com/blog/postgres-full-text-search-vs-the-rest).

**JSON API generation**

Use Postgres to [**generate JSON in the database**](https://www.amazingcto.com/sql-vs-graphql-performance-for-server-development/), write no server side code and directly give it to the API.

**Auditing**

Use Postgres with auditing with [**pgaudit**](https://github.com/pgaudit/pgaudit)

**GraphQL delivery**

Use Postgres with a [**GraphQL adapter**](https://graphjin.com/) to deliver GraphQL if needed.

**Vector database**

Use Postgres with [**pgvector**](https://github.com/pgvector/pgvector) for embeddings and AI similarity search instead of specialized vector databases.

**Session store**

Use Postgres for web session management instead of Redis with proper indexing on session IDs and expiration timestamps. Use [**hstore**](https://www.postgresql.org/docs/current/hstore.html) for a key value store. s **Rate limiting**

Use Postgres for API [**rate limiting**](https://neon.com/guides/rate-limiting) with atomic updates and time-based windows instead of Redis counters.
