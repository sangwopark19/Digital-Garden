Claude Code는 터미널 기반 코딩 에이전트로, 지속적인 에이전트 루프를 실행합니다. 사용자의 요청을 읽고, 수행할 작업을 추론하고, 도구를 호출하고, 결과를 관찰하고, 작업이 완료되거나 사용자의 입력이 필요할 때까지 이 과정을 반복합니다.

## 에이전트 루프

모든 상호작용은 동일한 기본 순환 과정을 따릅니다.

이 루프는 전적으로 터미널 프로세스 내에서 실행됩니다. 원격 실행 서버는 사용되지 않으며, 파일, 셸 및 자격 증명은 도구가 명시적으로 전송하지 않는 한(예: 또는 MCP 서버) 사용자의 컴퓨터를 벗어나지 `WebFetch` 않습니다 `WebSearch`.

각 대화가 시작될 때, 클로드 코드(Claude Code)는 모든 API 호출 앞에 추가될 두 개의 컨텍스트 블록을 생성합니다.

- 시스템 컨텍스트
- 사용자 컨텍스트

`getSystemContext()` in 에서 조립함 `context.ts`. 구성품:
- **Git 상태** — 현재 브랜치, 기본/메인 브랜치, Git 사용자 이름, `git status --short` 출력(2,000자보다 길면 잘림), 그리고 최근 5개 커밋을 표시합니다 `git log --oneline`.
- **캐시 파괴 주입** — 디버깅 중에 서버 측 프롬프트 캐시를 무효화하기 위해 내부적으로 사용되는 선택적 임시 문자열입니다.
`CLAUDE_CODE_REMOTE=1` Git 상태는 (원격/클라우드 모드)로 설정되어 있거나 설정에서 Git 명령어가 비활성화된 경우 건너뜁니다.

두 컨텍스트 블록 모두 대화가 진행되는 동안 **메모이제이션** `lodash/memoize` 됩니다. 호출하면 `setSystemPromptInjection()` 캐시가 즉시 지워집니다.

## 도구 실행 모델

Claude Code는 기본적으로 도구 호출을 자율적으로 실행하지 않습니다. 각 도구에는 `checkPermissions` 메서드가 있으며, 그 결과에 따라 다음에 수행될 작업이 결정됩니다.

| 권한 결과 | 무슨 일이 일어나는가 |
| --- | --- |
| `allow` | 도구가 즉시 실행되고 결과가 대화에 추가됩니다. |
| `ask` | Claude Code는 일시 중지 후 확인 대화 상자를 표시합니다. |
| `deny` | 툴 호출이 거부되었습니다. Claude는 오류 결과를 받았습니다. |

권한 동작은 활성화된 [권한 모드](https://www.mintlify.com/VineeTagarwaL-code/claude-code/concepts/permissions) 와 구성된 허용/거부 규칙에 따라 제어됩니다. `bypassPermissions` 모든 검사가 건너뛰어지는 모드도 있습니다. `acceptEdits` 파일 편집 도구는 자동으로 승인되지만, bash 명령은 여전히 확인 메시지를 표시합니다. 안전하고 읽기 전용인 도구 호출(예: `Read`, `Glob`, `Grep`)은 일반적으로 모든 모드에서 자동으로 승인됩니다.

## 대화형 모드와 비대화형(작업) 모드

## 대화형(REPL) 모드

기본 환경입니다. Claude Code는 React/Ink를 사용하여 실시간 터미널 UI를 렌더링합니다. 에이전트가 작동하는 동안 스트리밍 출력, 도구 사용 확인 및 스피너 애니메이션을 볼 수 있습니다. 메시지는 세션을 종료할 때까지 유지됩니다.

## 비대화형/인쇄 모드

표준 입력(stdin)을 직접 `--print` 또는 파이핑하여 활성화합니다. 사용자 인터페이스(UI)는 표시되지 않습니다. 출력은 표준 출력(stdout)으로 전송되므로 스크립트나 CI 파이프라인에서 캡처할 수 있습니다. 일회성 자동화 작업에 유용합니다.

### 하위 에이전트(작업 도구)

`Task` Claude는 도구( ) 를 통해 하위 에이전트를 생성할 수 있습니다 `AgentTool`. 각 하위 에이전트는 격리된 대화와 선택적으로 제한된 도구 세트를 사용하여 자체적인 중첩 에이전트 루프를 실행합니다. 하위 에이전트는 로컬(프로세스 내) 또는 원격 컴퓨팅 환경에서 실행될 수 있습니다. 하위 에이전트가 완료되면 결과가 상위 에이전트로 반환됩니다. 자세한 내용은 [도구(Tools)를](https://www.mintlify.com/VineeTagarwaL-code/claude-code/concepts/tools) 참조하십시오.

## 대화 저장 및 재개

대화 내용은 기본적으로 디스크의 JSON 형식의 대화록 파일로 저장됩니다 `~/.claude/`. 각 대화에는 고유한 세션 ID가 부여됩니다. 이전 대화를 이어서 진행하거나 `--resume <session-id>`, 목록에서 대화를 선택할 수 있습니다 `--resume`. 대화가 재개될 때:
- 전체 메시지 기록은 디스크에서 불러옵니다.
- 메모리 파일이 다시 발견되었으며, 대화가 처음 시작되었을 때와 다를 수 있습니다.
- 권한 모드는 세션에 저장되지 않은 경우 구성된 기본값으로 재설정됩니다.

긴 대화 내용은 주기적으로 **압축** 됩니다. 가장 오래된 메시지부터 요약하여 컨텍스트 창을 관리하기 쉽게 유지합니다. 전체 원본 대화 내용은 항상 디스크에 보존되며, 압축은 API로 전송되는 내용에만 적용됩니다.

## 쿼리 엔진

내부적으로 에이전트 루프의 각 "턴"은 **쿼리** 에 의해 구동됩니다. 이 쿼리는 `query.ts` 현재 메시지 목록을 Anthropic API로 전송하고 응답을 스트리밍 방식으로 반환합니다. 쿼리 엔진은 다음을 처리합니다.
- 토큰 출력을 터미널에 실시간으로 스트리밍합니다.
- `tool_use` 블록을 해당 도구 담당자에게 전달합니다.
- 턴별 토큰 및 도구 호출 예산을 시행합니다.
- 도구 결과를 수집하여 다음 모델 호출 전에 추가합니다.
- 컨텍스트 창이 가득 차면 압축을 시작합니다.
각 도구에는 `maxResultSizeChars` 속성이 있습니다. 결과가 이 제한을 초과하면 콘텐츠가 임시 파일에 저장되고 모델은 파일 경로가 포함된 미리보기를 수신하여 대용량 출력으로 인한 컨텍스트 창 오버플로를 방지합니다.
